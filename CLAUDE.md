This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A D&D 5e adventure creation and management system for writing, validating, and publishing adventures. Content is in Italian. Adventures are Markdown-based and packaged as PDF + ZIP releases.

> **Maps and dungeon generation** are in the separate repository **[dnd-maps](https://github.com/dracoroboter/dnd-maps)**.

## Common Commands

```bash
# Install dependencies (pandoc, wkhtmltopdf, zip, python3, Node.js, Playwright)
bash tech/scripts/setup.sh

# Scaffold a new adventure from the template
bash tech/scripts/new-adventure.sh <AdventureName>

# Interactively set/update an adventure's README metadata
python3 tech/scripts/adventure-wizard.py <AdventureName>

# Validate an adventure's structure and generate a report in tech/reports/
python3 tech/scripts/check-adventure.py <AdventureName>

# Build PDF + ZIP release artifact in releases/<AdventureName>/
./tech/scripts/release.sh <AdventureName> <version>

# Calculate D&D 5e encounter difficulty
python3 tech/scripts/encounter-difficulty.py -p 4 5 -m 1 5 -m 2 2

# Build a balanced encounter from SRD monster database
python3 tech/scripts/encounter-builder.py --party 4 3 --difficulty hard

# Create an NPC character sheet interactively
python3 tech/scripts/new-npc.py

# Generate stat blocks (NPC markdown → FightClub XML + PDF + PNG)
python3 tech/fightclub/generate-statblocks.py <AdventureName>

# Generate adventure PDF (cover + modules + stat blocks)
python3 tech/create-pdf-adventure/create-pdf-adventure.py <AdventureName>
```

## Architecture

### Adventure Structure

Each adventure lives in `adventures/<AdventureName>/` and must contain:

```
adventures/AdventureName/
├── README.md              # Public description — no spoilers
├── AdventureBook.md       # DM-facing notes (Kiro instructions)
├── PlanBook.md            # DM checklist & progress tracker
├── NomeAvventura.md       # Main narrative document
├── img/                   # Cover image (NomeAvventura_COVER.png)
├── maps/                  # Map images + map description .md files
├── characters/
│   ├── markdown/          # NPC_*.md source files
│   ├── fightclub/         # Generated FightClub XML
│   ├── statblock/         # Generated PDF/PNG stat blocks
│   └── img/               # Character artwork
└── 01_NomeModulo/
    └── NomeModulo.md      # At least one module required
```

`adventures/AdventureTemplate/` is the canonical scaffold used by `new-adventure.sh`. `adventures/FuoriDaHellfire/` is the reference for a fully-normalized adventure.

### Naming Conventions

See `tech/rules/adventure-template.md` § "Convenzioni di naming" — single source of truth.

Summary: PascalCase for adventure content, kebab-case for tech docs and scripts, UPPER_SNAKE_CASE for meta/project files. Code comments in English, everything else in Italian.

### Tooling

- **tech/scripts/**: All automation — Python 3 and Bash; no test framework
- **tech/rules/**: Specification documents (structure, content rules, glossary, NPC format)
- **tech/how-to/**: Step-by-step guides for common workflows
- **tech/fightclub/**: NPC markdown → FightClub XML + stat block PDF/PNG pipeline
- **tech/create-pdf-adventure/**: Adventure PDF generator (CSS + weasyprint)
- **tech/data/**: SRD 5.1 compendium (FightClub XML + XSD) and monster database (JSON)
- **tech/reports/**: Auto-generated by `check-adventure.py`; do not edit manually
- **releases/**: Generated artifacts; do not edit manually

### Saga Context

Active adventures belong to the "Lo Scettro di Tyr" saga:
- **LAnelloDelConte** — puntata 1 (normalized, in `adventures/`)
- **FuoriDaHellfire** — one-shot continuation of Hellfire Club starter set (normalized, in `adventures/`)
- **IlReSpezzato** — puntata 2 (pending migration from `.odt` in `legacy/`)
- **LoScettroDityr** — puntate 3–6 (A/B/C/D, pending migration)

Legacy `.odt` files are in `legacy/` and are read-only reference material.
