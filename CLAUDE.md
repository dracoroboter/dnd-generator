This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A D&D 5e adventure creation and management system for writing, validating, and publishing adventures. Content is in Italian. Adventures are Markdown-based and packaged as PDF + ZIP releases.

## Common Commands

```bash
# Install dependencies (pandoc, wkhtmltopdf, zip, python3, Playwright)
bash tech/scripts/setup.sh

# Scaffold a new adventure from the template
bash tech/scripts/new-adventure.sh <AdventureName>

# Interactively set/update an adventure's README metadata
python3 tech/scripts/adventure-wizard.py <AdventureName>

# Validate an adventure's structure and generate a report in tech/reports/
python3 tech/scripts/check-adventure.py <AdventureName>

# Build PDF + ZIP release artifact in releases/<AdventureName>/
./tech/scripts/release.sh <AdventureName> <version>

# Calculate D&D 5e encounter difficulty
# -p <count> <level> for each player group; -m <count> <CR> for each monster group
python3 tech/scripts/encounter-difficulty.py -p 4 5 -m 1 5 -m 2 2

# Create an NPC character sheet interactively
python3 tech/scripts/new-npc.py

# Generate a procedural dungeon map (BSP tree)
python3 tech/scripts/generate-dungeon.py --seed 42 --rooms 10 --output map.png

# Convert dungeon JSON to various SVG styles (blueprint, kenney, oldschool, iso, stone)
python3 tech/scripts/json-to-svg-blueprint.py <input.json> <output.svg>

# Batch-generate Watabou dungeon/city maps (requires Node.js + Playwright)
node tech/scripts/generate-watabou-dungeon-batch.js --count 5
```

## Architecture

### Adventure Structure

Each adventure lives in `adventures/<AdventureName>/` and must contain:

```
adventures/AdventureName/
├── README.md              # Public description — no spoilers
├── AdventureBook.md       # DM-facing notes (Kiro instructions)
├── PlanBook.md            # DM checklist & progress tracker
├── NomeAvventura.md       # Main narrative document
├── maps/
│   └── MappaGenerale.md   # Location index + map images
├── characters/
│   └── NPC_NomePersonaggio.md
└── 01_NomeModulo/
    └── NomeModulo.md      # At least one module required
```

`adventures/AdventureTemplate/` is the canonical scaffold used by `new-adventure.sh`. `adventures/AvventuraDiProva/` is the reference for a fully-normalized 3-module one-shot.

### Naming Conventions

- Markdown files: `PascalCase` (e.g., `NomeAvventura.md`, `NPC_IlConte.md`)
- Fixed names: `README.md`, `AdventureBook.md`, `PlanBook.md`
- Directories: `lowercase/` except module folders
- Module directories: `NN_NomeModulo/` (two-digit numeric prefix + PascalCase)
- NPC files: `NPC_<NomePersonaggio>.md`

### Tooling

- **tech/scripts/**: All automation — Python 3 and Bash; no test framework
- **tech/rules/**: Specification documents (structure, content rules, glossary, map pipeline)
- **tech/how-to/**: Step-by-step guides for common workflows
- **tech/templates/**: JSON schemas for dungeon objects (beds, doors, chests, etc.) used by map generators
- **tech/assets/tilesets/dcss/**: CC-licensed Dungeon Crawl Stone Soup tiles for `generate-dungeon.py`
- **tech/reports/**: Auto-generated by `check-adventure.py`; do not edit manually
- **releases/**: Generated artifacts; do not edit manually

### Map Generation Pipeline

1. **Procedural** — `generate-dungeon.py` (BSP tree → PNG/JSON)
2. **SVG styling** — `json-to-svg-*.py` in multiple visual styles (blueprint, kenney, oldschool, iso, stone)
3. **Browser automation** — Watabou generators (`generate-watabou-*.js`) via Playwright
4. **Manual refinement** — Upload PNG to Gemini with ambientazione prompt for polished final art

### Saga Context
